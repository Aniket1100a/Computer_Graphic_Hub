DDA Algorithm (Digital Differential Analyzer)

DDA is a basic line drawing algorithm that calculates intermediate points between two given points using linear interpolation. It divides the line into equal intervals along the major axis and then computes the corresponding points on the minor axis using slope. While easy to implement, DDA involves floating-point arithmetic, making it relatively slower compared to integer-based algorithms like Bresenham's Line Algorithm.

Bresenham's Line Drawing Algorithm

Bresenham's Line Algorithm is an efficient method for drawing lines on a grid. It avoids floating-point arithmetic by using integer operations, making it faster than DDA. This algorithm calculates the decision parameter to choose the next pixel to plot. It works well for lines with any slope and handles diagonal lines accurately.

Decision Parameter: ( P_k = 2 cdot text{{dy}} - text{{dx}} ) where ( text{{dy}} ) is the change in y-coordinate and ( text{{dx}} ) is the change in x-coordinate.

Step 1: Input

Get input coordinates (x1, y1) and (x2, y2) for the two endpoints of the line.

Step 2: Calculate Differences

Calculate the differences between the x-coordinates (Δx = x2 - x1) and y-coordinates (Δy = y2 - y1) of the two endpoints.

Step 3: Determine Initial Decision Parameter

Calculate the initial decision parameter (P0 = 2Δy - Δx).

Step 4: Draw Pixels

Inside a loop (from x = x1 to x = x2):
- Plot the pixel at coordinates (x, y).
- If Pk < 0, update Pk and y. If Pk ≥ 0, only update Pk.
